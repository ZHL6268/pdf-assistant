# Phase 8 Plan

## 目标

第八阶段聚焦收尾、稳定性与交付准备，目标是让当前 MVP 从“核心功能可用”推进到“日常测试和演示更稳定、文档更完整、假交互更少”。

## 范围

1. 收口假交互
   - 将仍未实现的按钮、页签和入口改成明确的只读状态
   - 避免继续产生误导性的点击反馈和测试噪音

2. 完善输入和错误体验
   - 清理默认 demo 表单值
   - 优化认证和聊天等关键路径的基础可用性

3. 做一轮工程清洁
   - 移除无价值的 demo 残留
   - 维持代码、文档和目录结构的清晰度

4. 补齐文档
   - 同步 README、Architecture、Frontend Baseline
   - 写清楚当前 MVP 已完成范围和仍未实现边界

## 非目标

- 不新增新的核心业务能力
- 不引入向量检索或多文档检索
- 不迁移到 Next.js

## 完成标准

- 主要未实现入口都已转成清晰的只读或禁用状态
- 当前核心链路测试噪音明显减少
- 文档与代码状态一致
- 仓库保持可构建、可运行、可继续迭代
