# PRD

## 产品定位

AI PDF Assistant 是一个轻量级 AI SaaS 应用，面向需要快速理解 PDF 内容的用户。产品目标不是做一个单点演示，而是做出一个具备真实 SaaS 形态的 MVP，覆盖从认证、上传、摘要生成到问答回看的完整闭环。

目标用户主要包括：

- 学生和研究人员
  - 需要快速理解论文、报告和学习材料
- 知识型工作者
  - 需要快速提取商业文档、操作手册和内部资料中的关键内容
- 创业者与产品负责人
  - 需要一个可直接体现 AI 产品价值的文档助手

产品的核心价值：

- 降低用户阅读长文档的时间成本
- 提供围绕单一文档的结构化 AI 能力
- 形成可持续演进的 SaaS 产品基础

## 核心功能列表

MVP 范围内的核心功能如下：

1. 用户认证
   - 用户注册、登录、登出
   - 会话保持
   - 受保护页面访问控制

2. PDF 上传
   - 单文件上传
   - PDF 类型校验
   - 文件大小限制

3. 文档仪表盘
   - 展示用户上传的文档列表
   - 展示上传时间、处理状态、摘要状态

4. AI 摘要生成
   - 基于文档提取文本生成结构化摘要
   - 摘要结果持久化保存

5. 文档问答
   - 针对单个文档进行上下文问答
   - 回复基于该文档内容而非全局知识

6. 历史记录
   - 保存摘要结果
   - 保存对话历史
   - 支持回访用户已处理过的文档

MVP 明确不包含以下内容：

- OCR 扫描件识别
- 多文档联合检索
- 团队协作和多租户工作区
- 支付、订阅和计费
- 高级模型切换和自定义 Prompt
- 复杂检索增强和向量数据库优化

## 用户使用流程

用户主流程如下：

1. 用户进入 landing page
2. 用户注册或登录
3. 用户进入受保护的 dashboard
4. 用户上传 PDF
5. 系统完成文件校验、存储和文本提取
6. 文档出现在 dashboard 列表中
7. 用户进入 document detail 页面
8. 用户生成摘要并查看摘要结果
9. 用户提出与该文档相关的问题
10. 系统返回 grounded answer 并保存对话历史
11. 用户后续可再次进入该文档查看摘要和聊天记录

页面范围包括：

- `/`
  - Landing page
- `/login`
  - 登录页
- `/signup`
  - 注册页
- `/dashboard`
  - 文档仪表盘
- `/documents/[id]`
  - 文档详情页，包含摘要与聊天区域
